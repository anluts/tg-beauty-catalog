# Research: Telegram Mini Apps для бьюти-каталога

> Исследование проведено: 2026-04-07  
> Контекст: разработка каталога услуг бьюти-мастера как Telegram Mini App

---

## 5 примеров Telegram Mini Apps в нише бьюти и смежных нишах

---

### 1. DIKIDI Online (`@dikidi_bot`)

**Что делает:** Крупнейший маркетплейс записи к мастерам красоты в России/СНГ. 2M+ зарегистрированных мастеров: нейл-мастера, парикмахеры, лешмейкеры, косметологи, массажисты. Полноценный Mini App для поиска, просмотра портфолио и онлайн-записи.

**Экраны:**
- Главная: выбор города + иконки категорий (маникюр, волосы, брови/ресницы, лицо, тело, барбер)
- Список мастеров/салонов: переключатель карта/список
- Профиль мастера: фото портфолио, цены, адрес, рейтинг, отзывы
- Запись: выбор услуги → выбор мастера → календарь → слоты времени
- Мои записи: предстоящие / прошедшие
- Уведомления: напоминание за 24ч и за 1ч

**Сильные стороны:**
- Переключатель карта/список для поиска ближайших салонов — отличная UX-находка
- Богатые страницы салонов: обложка, рейтинг, выдержки отзывов, список услуг
- Push-уведомления через Telegram (нативно)
- Поддержка нескольких мастеров в одном салоне с выбором конкретного
- Вкладка "Мои записи" как чистый таймлайн

**Слабые стороны:**
- Приложение — обёрнутое PWA, не нативный Mini App. Нет `MainButton`, `BackButton`, `HapticFeedback`
- Повторная верификация телефона — несмотря на то что Telegram уже знает номер
- Медленная загрузка карты на мобильном
- Глубокая таксономия категорий (5+ уровней) — навигационный ад
- Нет фото-отзывов
- Нет "мгновенной записи" — каждое бронирование требует подтверждения мастера
- Главный экран generic — нет персонализации по истории посещений

**Что мы можем сделать лучше:**
- Блок "Ваши недавние мастера" на главном экране — быстрое повторное бронирование
- Фото-отзывы (главный конкурентный пробел)
- "Instant Book" как фильтр: только мастера без подтверждения
- Максимум 2 уровня категорий
- Нативная интеграция через Telegram Login (без переввода телефона)

### 2. YClients (платформа + `@yclients_bot`)

**Что делает:** №1 SaaS для управления салонами красоты в России (100,000+ бизнесов). Виджет онлайн-записи, который можно открыть как Mini App через inline-кнопку в боте. Охватывает парикмахерские, нейл-студии, спа, барбершопы.

**Экраны:**
- Выбор категории услуги (Волосы, Ногти, Брови, ...)
- Сетка мастеров: аватар, имя, специализация, теги
- Выбор услуги: список с категориями, длительность, цена
- Календарь + слоты времени
- Форма клиента (имя, телефон)
- Экран подтверждения с кодом записи

**Сильные стороны:**
- Глубокий каталог услуг с иерархией: категория → подкатегория → услуга
- Реальная доступность мастеров в реальном времени (синхронизировано с расписанием)
- Линейный понятный wizard записи — низкий процент ошибок
- Широкое распространение: тысячи бизнесов на платформе

**Слабые стороны:**
- Виджет создан НЕ для Telegram — это responsive-сайт в iframe. Не использует ни один Telegram API
- Нет нативных цветов темы Telegram — выглядит чужеродно
- Загрузка 3–5 секунд (тяжёлый бандл виджета)
- Нет интеграции с профилем Telegram — нужно вводить имя и телефон каждый раз
- Нет follow-up в чате после записи (упущенная CRM-возможность)
- Кнопки используют DOM, не `MainButton` API

**Что мы можем сделать лучше:**
- Нативная сборка с Telegram SDK: `tg.MainButton`, `tg.BackButton`, `tg.themeParams`
- Предзаполнение имени и телефона из профиля Telegram (с разрешения)
- Подтверждение записи прямо в чате Telegram
- Загрузка < 1.5 секунды
- Сохранение клиента в CloudStorage — не вводить данные повторно

### 3. Hamster Kombat (игра, 100M+ пользователей)

**Что делает:** Клик-игра с виртуальной экономикой. Не бьюти, но задал стандарты UX для Mini Apps. Карточная сетка "магазина улучшений" — прямой аналог каталога услуг.

**Экраны:**
- Главный: большой объект для тапа + прогресс энергии
- Магазин улучшений: карточная сетка с категориями
- Друзья / реферальная программа
- Earn: задачи и награды
- Кошелёк / вывод средств
- Таблица лидеров

**Сильные стороны:**
- Время загрузки < 1 секунды — perceived performance-эталон
- Карточный грид магазина: иконка + название + стоимость + одно нажатие для действия — прямо применимо к каталогу услуг
- Горизонтальный скролл табов-категорий — стало стандартом навигации по категориям в Mini Apps
- Bottom navigation bar с 5 иконками — де-факто стандарт Mini App-навигации
- Haptic feedback на каждый тап — ощущение нативности
- Индикаторы прогресса везде — хороший образец визуальной обратной связи

**Слабые стороны:**
- Нет поиска — только категориальный браузинг
- Нет детальных страниц — всё решается одним тапом на карточку
- Нет описаний, нет фото-галерей, нет отзывов
- Мелкие тач-таргеты в некоторых местах

**Что мы можем сделать лучше:**
- Взять grid + bottom nav + категориальные табы, но добавить полноценные детальные страницы с галереей и описанием на каждую услугу
- Добавить поиск/фильтр над грид-ом

### 4. Wallet (`@wallet`) — TON/криптовалютный кошелёк

**Что делает:** Официальный Telegram-кошелёк для TON, USDT и Bitcoin с P2P-обменом. Не бьюти, но лучший пример wizard-навигации для сложных flow в Mini Apps.

**Экраны:**
- Дашборд с балансом и быстрыми действиями
- Wizard покупки криптовалюты (пошаговый)
- P2P биржа: список предложений → детали → чат с контрагентом
- История транзакций с фильтрами
- QR-код получения/отправки
- KYC-верификация

**Сильные стороны:**
- Пошаговый wizard для сложных flow (Купить → Сумма → Способ оплаты → Подтвердить → Готово) — идеальный шаблон для flow записи
- Цветовые статус-чипы на транзакциях: Pending / Completed / Failed — применимо для статусов записей
- Bottom sheet модалки для вторичных действий — плавные переходы без jarring-навигации
- Чёткая коммуникация ошибок с кнопкой повтора

**Слабые стороны:**
- Онбординг сложен и требует знания крипто
- TON-only ограничивает аудиторию
- Страница настроек перегружена опциями

**Что мы можем сделать лучше:**
- Применить wizard-паттерн для записи: Услуга → Мастер → Дата/Время → Подтверждение → Готово
- Статус-чипы для состояний записи: Подтверждена / Ожидает / Выполнена / Отменена

### 5. AliExpress Telegram Mini App (e-commerce каталог)

**Что делает:** Каталог товаров с корзиной, поиском и оформлением заказа. Ближайший аналог к каталогу услуг по структуре: листинг → детальная страница → корзина → чекаут.

**Экраны:**
- Главная: баннеры + иконки категорий
- Поиск с фильтрами (цена, рейтинг, доставка)
- Детальная страница товара: свайпабельная галерея, вкладки описания, продавец, отзывы
- Корзина с управлением количеством
- Wizard чекаута (адрес → время → оплата → подтверждение)
- Отслеживание заказа

**Сильные стороны:**
- Swiper галерея на детальной странице — эталон для показа портфолио услуг
- "Добавить в корзину" закреплена снизу — всегда видна
- Секция отзывов: звёздный гистограмм + фото-отзывы + рейтинг по параметрам
- "Похожие товары" внизу детальной страницы

**Слабые стороны:**
- Тяжёлый — медленная загрузка на 3G/4G
- Чекаут уводит из Telegram для оплаты
- Спамные уведомления

**Что мы можем сделать лучше:**
- Оставить паттерн: галерея + прайс + CTA + отзывы + рекомендации
- Платёж через нативный Telegram Payments API — без выхода из приложения
- Фото-отзывы для услуг мастера (никто из бьюти-конкурентов это не делает)

## Обязательные UI-элементы

### Навигация
- **Bottom tab bar** (4–5 иконок): Каталог / Мастера / Мои записи / Профиль — стандарт Mini Apps
- **Горизонтальный скролл категорий** (chips) под поиском — не дропдаун
- **`tg.BackButton`** — нативная кнопка назад Telegram, не DOM-кнопка
- **`tg.MainButton`** — основной CTA (Записаться, Оплатить, Подтвердить)

### Каталог
- Поисковая строка (закреплена сверху)
- 2-колоночный карточный грид
- Карточка услуги: фото + название + цена + длительность + рейтинг
- Skeleton-загрузка (не спиннер)
- Пустое состояние с иллюстрацией при нулевых результатах

### Детальная страница
- Свайпабельная фото-галерея (3–5 фото)
- Цена (крупно, "от X₽" если переменная)
- Длительность-бейдж
- Рейтинг + количество отзывов
- Блок "Что входит"
- Карточка мастера (кликабельная → профиль)
- Горизонтальный скролл отзывов
- **Фото-отзывы** (ключевой дифференциатор)

### Wizard записи
- Прогресс-бар (не точки — показывает процент)
- Шаг 1: Календарь (недоступные даты серые, доступные подсвечены)
- Шаг 2: Слоты времени кнопками в сетке (не нативный time input)
- Шаг 3: Данные (предзаполнены из Telegram профиля)
- `MainButton` = "Далее" (неактивна до выбора)
- Back button = предыдущий шаг

### Подтверждение
- Большая иконка успеха (с анимацией)
- Номер записи
- Сводка: мастер, услуга, дата/время, адрес
- "Добавить в календарь"
- "Поделиться с другом" (нативный Telegram share)
- `MainButton` закрывает Mini App → возвращает в чат

## Что мы можем сделать лучше конкурентов

### Критические пробелы конкурентов

1. **Нативная интеграция Telegram** — ни один бьюти-конкурент не использует `MainButton`, `BackButton`, `HapticFeedback` и `themeParams` корректно. Это одно делает UX принципиально лучше.

2. **Запись нескольких услуг одновременно** — ни Yclients, ни Dikidi не поддерживают корзину: нельзя записаться на маникюр + педикюр за один раз. Огромный функциональный пробел.

3. **Фото-отзывы** — Dikidi имеет текстовые отзывы, никто не показывает фото-отзывы в приложении. Это конверсионный драйвер (+15–20% по e-commerce бенчмаркам).

4. **Персонализированный главный экран** — "Добро пожаловать, Александра — ваши недавние мастера:" с CTA повторной записи. Ни одно приложение этого не делает.

5. **Скорость загрузки** — Yclients: 3–5с, Dikidi: 2–4с. Цель: < 1.5 секунды.

### Уникальные функции для реализации

| Функция | Почему важна |
|---------|-------------|
| "Умное повторное бронирование" | Если последний визит ~4 недели назад — предложить записаться снова к тому же мастеру |
| Корзина (мультиуслуга) | Маникюр + педикюр за одну сессию = один поток записи |
| "Удивите меня" | Установить бюджет + категорию → первый доступный слот у любого мастера |
| Социальное доказательство | "3 человека записались к этому мастеру сегодня" (FOMO, в бьюти-приложениях отсутствует) |
| Нативный share мастера | "Поделиться мастером" → красивая preview-карточка прямо в любой чат Telegram |
| Бесплатная отмена | Одним тапом в чате до X часов до визита |

## Технические требования для нативного ощущения

```
Telegram.WebApp.ready()          // скрыть стандартный индикатор загрузки
Telegram.WebApp.expand()         // полноэкранный режим при открытии
tg.themeParams                   // читать цвета темы, не хардкодить
tg.MainButton                    // основной CTA — не DOM-кнопка
tg.BackButton                    // нативная кнопка назад
tg.HapticFeedback.impactOccurred // тактильный отклик при подтверждении
tg.HapticFeedback.notificationOccurred("error") // при ошибке оплаты
tg.showConfirm()                 // перед отменой записи
tg.sendData()                    // отправить данные записи в бота
```

### Цели производительности

| Метрика | Цель |
|---------|------|
| Time to Interactive | < 1.5 секунды |
| Рендер каталога (20 карточек) | < 500ms |
| Переход между шагами wizard | < 200ms |
| Размер начального бандла | < 200KB |

## Антипаттерны, которых нужно избегать

| Антипаттерн | Проблема |
|-------------|---------|
| Кастомная форма входа/авторизации | Пользователь уже авторизован через Telegram |
| `<input type="date">` или `<input type="time">` | Выглядит ужасно на мобильных устройствах |
| Hamburger-меню | Не идиоматично для Mini Apps — нужен bottom tab bar |
| `alert()` / `confirm()` браузерные диалоги | Игнорируют тему Telegram, ломают UX |
| DOM-кнопка в самом низу экрана | Конкурирует с `MainButton` |
| Перезагрузка страницы при навигации | Теряет состояние, ощущается медленным |
| Хардкод цветов | Ломается при смене тёмной/светлой темы Telegram |
| Поток записи > 4 шагов | Высокий процент отказов |
| Нет skeleton-загрузки | Белая вспышка перед контентом |

## Экспертная оценка

> Три независимых эксперта проанализировали концепцию приложения на основе данных исследования.

---

### Взгляд 1: UX-дизайнер

#### Топ-5 ошибок новичков

**1. Игнорирование данных Telegram-профиля**
Новички делают форму с полями «Имя», «Телефон» — точно как на обычном сайте. Пользователь думает: «А зачем я заходил через Telegram?». `initDataUnsafe.user` даёт `first_name`, `last_name`, `username`, `photo_url` бесплатно. Подставляй автоматически, давай только подтвердить.

**2. Глубокая иерархия категорий**
Копируют структуру Dikidi: «Услуги → Ногти → Маникюр → Покрытие → Гель-лак». На 4-м уровне пользователь нажимает «назад» 4 раза и закрывает приложение. Максимум 2 уровня: горизонтальные табы + карточки услуг.

**3. Игнорирование нативных паттернов Telegram**
Делают приложение как обычный React-сайт — роутер с переходами, хедер с логотипом, footer. Пользователь нажимает физическую кнопку «назад» на Android → Telegram закрывает всё приложение целиком. Детали услуги — в Bottom Sheet. Навигация — через `tg.BackButton`, а не `history.back()`.

**4. Отсутствие Skeleton UI**
Белый экран или спиннер на 2–3 секунды в контексте Telegram, где всё мгновенно = «приложение сломалось». CTR после первого открытия падает на 40–60%. Skeleton-карточки должны появляться мгновенно, реальный контент подгружается поверх.

**5. Время записи без контекста услуги**
Показывают сетку слотов без объяснения длительности. Пользователь выбирает 14:00, мастер отменяет — оказывается, маникюр + педикюр занимают 3 часа, и не успевает. Показывай длительность ДО выбора времени. Недоступные слоты объясняй тултипом.

#### Что обязательно должно быть

| Приоритет | Элемент | Почему критично |
|----------|---------|----------------|
| 🔴 P0 | Автоподстановка данных из Telegram | Первое впечатление не перебить |
| 🔴 P0 | Skeleton UI при загрузке | Без этого выглядит сломанным |
| 🔴 P0 | `MainButton` для бронирования | Нативный паттерн — без него ломается UX |
| 🔴 P0 | Подтверждение в бот после записи | Без этого пользователь не верит, что запись прошла |
| 🟠 P1 | `HapticFeedback` на ключевых действиях | Ощущение нативности |
| 🟠 P1 | `BackButton` на всех вложенных экранах | Иначе закроет всё приложение |
| 🟠 P1 | Горизонтальный скролл категорий (не дропдаун) | Глубокая иерархия убивает навигацию |
| 🟡 P2 | Фото работ мастера в отзывах | Конкурентное преимущество (никто не делает) |
| 🟡 P2 | «Записаться снова» в истории записей | Retention и повторные визиты |
| 🟡 P2 | «Добавить в календарь» на экране успеха | Долгосрочный retention |

#### Что больше всего раздражает пользователей

- **Выбор даты без шапки с контекстом** — пользователь на шаге 2 не помнит, кого и на что записывается. Всегда держи имя мастера и услугу в sticky-хедере.
- **Мелкие кнопки слотов времени** — 24 кнопки 40×30px, промахивается, злится. Минимум 48×48px, 3 слота в ряд.
- **Отсутствие прогресс-индикатора** — «Шаг 2 из 3» в хедере wizard обязателен. Без него пользователь не знает, сколько ещё.
- **Обязательный ввод телефона до просмотра каталога** — 60–70% уходят. Запрашивай только в момент бронирования через нативный `requestContact()`.
- **Нет кнопок «Перенести» / «Отменить»** в разделе «Мои записи» — разрушает доверие к цифровому инструменту.
- **«Занятый» слот полностью скрыт** — пустая сетка пугает («мастер вообще работает?»). Показывай занятые слоты серыми — это создаёт контекст «мастер востребован».
- **Confirm-диалог при закрытии приложения** — дай пользователю закрыть без вопросов. Данные черновика сохраняй молча в sessionStorage.

### Взгляд 2: Маркетолог

#### Что заставляет записаться

Женщина не покупает стрижку — она покупает **версию себя после**. Любой элемент интерфейса должен продавать трансформацию, а не услугу.

**5 главных психологических триггеров:**

| Триггер | Описание | Как реализовать |
|---------|---------|----------------|
| Социальное зеркало | «Такие же женщины уже выбрали этого мастера» | Отзывы с именем, фото, конкретной услугой |
| Устранение страха ошибки | Beauty — высокорисковая покупка | Политика отмены видна ДО бронирования |
| Готовность мастера | «Слот свободен прямо сейчас» — ощущение привилегии | Конкретная дата ближайшего слота на главном экране |
| Лёгкость обратного хода | «Если что — перенесу» снижает барьер | «Перенос за 24ч — без вопросов» — в карточке мастера |
| Конкретность результата | «Покрытие держится 3–4 недели» вместо «маникюр гель-лак» | Язык результата во всём тексте |

#### Убийцы конверсии

1. **Экран выбора времени до формирования желания** — время выбирают ПОСЛЕ «хочу именно к этому мастеру». Ошибка: показывать слоты раньше, чем пользователь влюбился в портфолио.
2. **Прайс без обоснования** — «Маникюр 2500р» без фото и контекста всегда кажется завышенным.
3. **Любое лишнее поле** — каждое дополнительное поле убивает конверсию на 10–15%.
4. **Нет ответа на «а вдруг не понравится?»** — отсутствие политики отмены = молчаливый стоп-сигнал.
5. **Медленная загрузка портфолио** — фото грузятся 3+ сек → пользователь ушёл.
6. **Скучный экран подтверждения** — «Запись создана #4821» без эмоции = нет якоря для возврата.
7. **Невозможность выбрать несколько услуг** — маникюр + педикюр требует 3 отдельных захода → убивает средний чек.

#### Высокоэффективные элементы (по убыванию влияния)

**🥇 Фото «до/после» и портфолио — критическое влияние**
- Реальные работы, не стоковые (стоковые фото мгновенно разрушают доверие)
- Тэги на фото: «гель-лак, 2500р, держится 4 недели» → фото становится кликабельным каталогом
- Пометка «работа этой недели» создаёт ощущение активного мастера
- Видео-процесс 15 сек конвертирует на 40% лучше фото

**🥈 Социальное доказательство — очень высокое влияние**
- Отзывы с именем, аватаром и конкретной услугой — не анонимные «Анна К.»
- «Записалась 3 часа назад» — живой счётчик последней записи
- Фото-отзывы клиентки со своим результатом — конвертируют в 3x лучше текстовых
- Рейтинг 5.0/5 вызывает недоверие (слишком идеально) — пусть будет 4.8

**🥉 Стратегия подачи цены — высокое влияние**
- Якорный эффект: показывать сначала дорогой пакет, потом базовую услугу
- «Маникюр + педикюр = 3800р (педикюр в тот же день +1200р вместо 2000р)» — кросс-продажа встроена в цену
- Цена за результат: «Цвет, который держится месяц — 2500р» вместо «гель-лак 2 часа»

#### Первые 5 секунд: что должен увидеть пользователь

**Единственная цель первого экрана — создать желание, не показывать меню.**

```
[Фото мастера или лучшая работа этой недели — крупно]

«Привет! Я Алина — мастер маникюра в Москве»
⭐ 4.9  •  127 работ  •  Ближайшая запись: завтра в 14:00

[Кнопка: «Записаться» — на всю ширину]

[Горизонтальная карусель 5 лучших фото]

[«94 клиентки записались в апреле»]
```

- Одна кнопка CTA — «Записаться», не «Посмотреть услуги»
- Конкретная дата ближайшего слота убирает абстракцию
- Лицо мастера на первом экране создаёт личную связь
- Никакого онбординга — пользователь пришёл по ссылке, он уже «тёплый»

#### Стратегии повторного вовлечения

**Умные напоминания на основе цикла услуги:**
- Маникюр гель-лак: через 3 недели → пуш «Алина напоминает: через неделю ваш маникюр начнёт отрастать. Свободные слоты 👇»
- Конверсия таких напоминаний: 35–50% против 3–5% у обычных рассылок

**Система статусов (не «баллы» — а принадлежность):**
- «Постоянная клиентка» после 3 визитов
- «VIP» после 10 визитов — приоритетная запись
- «День рождения клиентки» — автоматическое поздравление за 5 дней + спецпредложение

**Реферальная механика:**
- «Пришла подруга по вашей ссылке → ваш следующий маникюр −300р»
- Реферальная ссылка генерируется в Mini App, шерится через Telegram одним тапом

#### Копирайтинг: язык, который конвертирует

| ❌ Не работает | ✅ Работает |
|---|---|
| «Маникюр гель-лак» | «Ногти, за которыми не надо следить месяц» |
| «Запишитесь на процедуру» | «Выберите удобное время» |

**Текст подтверждения записи** — момент максимальной лояльности:

### Взгляд 3: Telegram-разработчик

#### Ограничения платформы

**iOS vs Android WebView:**
- iOS (WKWebView): `position: fixed` при открытой клавиатуре «прыгает» и не возвращается; `100vh` ведёт себя непредсказуемо; `input[type="date"]` рендерит нативный iOS пикер — некрасивый и некастомизируемый
- Android: клавиатура сдвигает viewport, `window.innerHeight` меняется; на Android 9 и ниже WebView может быть сильно устаревшим

**Решение проблемы viewport:**
```javascript
// Слушать событие Telegram, а не CSS vh
window.Telegram.WebApp.onEvent('viewportChanged', ({ is_state_stable }) => {
  if (is_state_stable) {
    document.documentElement.style.setProperty(
      '--tg-viewport-stable-height',
      `${window.Telegram.WebApp.viewportStableHeight}px`
    );
  }
});

// В CSS использовать ТОЛЬКО эту переменную:
.screen { height: var(--tg-viewport-stable-height); /* НЕ 100vh */ }
```

**CloudStorage API — лимиты:**
- Максимум 1024 ключа, 4096 байт на ключ
- Асинхронный — нет синхронного чтения
- Операции выполнять последовательно, не параллельно (race condition)
- Хранить весь стейт в одном ключе `app_state` как JSON

#### Что работает отлично — использовать обязательно

**MainButton — главный CTA:**
```javascript
// Всегда снимать предыдущий обработчик перед добавлением нового!
useEffect(() => {
  const handler = () => handleBooking(selectedSlot);
  tg.MainButton.onClick(handler);
  return () => tg.MainButton.offClick(handler); // Cleanup!
}, [selectedSlot]);

// Disable вместо hide/show для wizard-шагов (нет мерцания):
selectedTime ? tg.MainButton.enable() : tg.MainButton.disable();
```

**HapticFeedback — делает приложение «живым»:**
```javascript
const Haptic = {
  selectService:    () => tg.HapticFeedback.selectionChanged(),
  bookingTap:       () => tg.HapticFeedback.impactOccurred('medium'),
  bookingSuccess:   () => tg.HapticFeedback.notificationOccurred('success'),
  bookingError:     () => tg.HapticFeedback.notificationOccurred('error'),
  gallerySwipe:     () => tg.HapticFeedback.impactOccurred('light'),
  dateSelect:       () => tg.HapticFeedback.selectionChanged(),
};
```

**themeParams — нативный вид в любой теме:**
```javascript
const applyTheme = () => {
  const root = document.documentElement;
  const t = tg.themeParams;
  root.style.setProperty('--tg-bg', t.bg_color);
  root.style.setProperty('--tg-text', t.text_color);
  root.style.setProperty('--tg-button', t.button_color);
  root.style.setProperty('--tg-secondary-bg', t.secondary_bg_color);
};
applyTheme();
tg.onEvent('themeChanged', applyTheme);
```

**Telegram Payments для депозитов:**
- Нативный UI — пользователь не покидает Telegram, конверсия выше
- Сохранённые карты, Apple Pay / Google Pay работают автоматически
- Снижает no-show на 70–80%
- Провайдеры: ЮKassa, Сбер, Тинькофф (~2.5% комиссия)

#### Что НЕ делать (антипаттерны разработчика)

| Антипаттерн | Проблема | Решение |
|-------------|---------|---------|
| `window.history.pushState` для навигации | BackButton Telegram не связан с browser history | Управлять стеком самостоятельно |
| Параллельные `CloudStorage` операции | Race condition, потеря данных | Только последовательно |
| `window.open()` для внешних ссылок | Блокируется или ломает UX | `tg.openLink()` или `tg.openTelegramLink()` |
| `alert()` / `confirm()` браузера | Игнорирует тему Telegram | `tg.showPopup()` / `tg.showConfirm()` |
| `BrowserRouter` вместо `HashRouter` | TMA не поддерживает серверный роутинг | `HashRouter` (URL вида `/#/service/1`) |
| Тяжёлые библиотеки | Moment.js 67KB, FullCalendar огромный | `dayjs` (2KB), кастомный календарь |
| `tg.ready()` до инициализации данных | MainButton мерцает | Вызывать `ready()` только после загрузки |

#### Рекомендуемый стек

```
React 18 + TypeScript + Vite
├── Zustand (стейт-менеджмент, 3KB)
├── TanStack Query v5 (кэширование запросов)
├── React Router v6 (hash mode!)
├── dayjs (работа с датами, 2KB)
└── Tailwind CSS v4
```

**Стратегия кэширования:**
```javascript
// Каталог услуг — долгое время жизни
staleTime: 5 * 60 * 1000   // 5 минут «свежие»

// Слоты времени — короткое время жизни
staleTime: 30 * 1000        // 30 секунд (слоты быстро занимаются)
refetchInterval: 30 * 1000  // Автообновление
```

**Оптимизация изображений:**
- Thumbnails: WebP 150×150, quality 70% — для каталога
- Preview: WebP 400×400, quality 80% — для карточки
- Full: WebP 800×800, quality 85% — для галереи
- Prefetch данных по `onTouchStart` — на 300ms раньше клика

#### Специфические баги, которые укусят

| Баг | Симптом | Решение |
|-----|---------|---------|
| `MainButton.onClick` дублируется | Кнопка срабатывает дважды | Всегда `offClick` перед `onClick` |
| Double-tap на iOS | `onClick` срабатывает 2 раза | Дебаунс `{ leading: true, trailing: false }` |
| `viewportHeight` после `expand()` | Старое значение (≈50% экрана) | Читать в `viewportChanged` с `is_state_stable` |
| Scroll position сбрасывается | При возврате каталог вверху | Сохранять в `sessionStorage` при уходе |
| Стейт wizard теряется | iOS выгружает JS при сворачивании | Сохранять wizard в `sessionStorage` при каждом изменении |
| Клавиатура на iOS | Input уходит под клавиатуру | `scrollIntoView` через `setTimeout(300)` после `focus` |
| Dark mode неопределён | `colorScheme === undefined` на старте | Fallback через `matchMedia('prefers-color-scheme: dark')` |
| Google Fonts | +200–400ms загрузки | Использовать системные шрифты: `-apple-system, system-ui` |

```

✓ BackButton управляет собственным навигационным стеком

✓ Роутинг в hash mode (#/)

✓ themeParams применён через CSS-переменные
✓ CloudStorage операции только последовательные
```
