## 1. Архитектура проекта

```
tg-beauty-catalog/
├── tg-app/                  # Весь фронтенд (деплоится на Vercel)
│   ├── index.html           # Точка входа, подключает все скрипты
│   ├── css/styles.css       # Все стили (1500+ строк)
│   └── js/
│       ├── data.js          # Все данные: мастер, услуги, отзывы
│       ├── tg.js            # Обёртка Telegram WebApp SDK
│       ├── state.js         # Глобальное состояние + localStorage
│       ├── router.js        # SPA роутер (стек экранов)
│       ├── onboarding.js    # Онбординг при первом запуске
│       ├── offer.js         # Оффер скидки 15% при первом запуске
│       ├── app.js           # Инициализация, таб-бар
│       └── screens/
│           ├── catalog.js   # Каталог услуг (главный экран)
│           ├── service.js   # Детальная страница услуги
│           ├── master.js    # Профиль мастера
│           ├── bookings.js  # Мои записи
│           ├── wizard.js    # Мастер записи (3 шага)
│           ├── search.js    # Поиск
│           ├── success.js   # Экран успешной записи
│           └── portfolio.js # Все работы мастера
├── .env.example             # Шаблон переменных окружения
├── brief.md                 # Техническое задание
└── TESTING.md               # Этот файл
```

**Стек:** Vanilla JS (IIFE-модули), без фреймворков и сборщиков. Один HTML файл, скрипты загружаются последовательно через `<script src>`. Порядок загрузки важен — data.js и tg.js должны быть перед остальными.

## 2. Локальный запуск

### Требования
- Python 3 (встроен в macOS) **или** Node.js **или** любой HTTP-сервер
- Telegram аккаунт (для тестирования в Mini App)

### Шаг 1 — Клонировать репозиторий

```bash
git clone https://github.com/anluts/tg-beauty-catalog.git
cd tg-beauty-catalog
```

### Шаг 2 — Запустить HTTP-сервер

```bash
# Вариант 1: Python
cd tg-app
python3 -m http.server 3000

# Вариант 2: Node.js (npx)
cd tg-app
npx serve -p 3000

# Вариант 3: PHP
cd tg-app
php -S 0.0.0.0:3000
```

### Шаг 3 — Открыть в браузере

```
http://localhost:3000
```

> ⚠️ Открывать нужно именно через HTTP-сервер, не через `file://`. Иначе не будет localStorage.

### Тестирование в браузере (без Telegram)

Приложение работает без Telegram — SDK автоматически возвращает fallback-данные:
- Имя пользователя: **Александра**
- Вместо Telegram BackButton — стрелка в шапке
- Вместо Telegram MainButton — синяя кнопка снизу
- Вместо haptic feedback — ничего (игнорируется)

## 3. Тестирование в Telegram

### Вариант A — через прямую ссылку (рекомендуется)

Открыть в Telegram: **https://t.me/oz_beauty_bot/app**

### Вариант B — через бот

1. Найти бота: `@oz_beauty_bot`
2. Нажать кнопку **«Записаться»** в нижней панели

### Вариант C — тестирование своего деплоя

Если нужно тестировать изменённый код:

1. Задеплоить на Vercel (см. раздел 6)
2. В BotFather: `/mybots` → `@oz_beauty_bot` → Bot Settings → Configure Mini App → Edit Mini App URL
3. Вставить новый URL
4. Открыть Mini App

> ⚠️ Telegram кэширует Mini App. Если изменения не отображаются — закрыть и открыть снова, или через 3 точки → Reload Page.

## 5. Известные особенности и ограничения

### MVP-ограничения (задокументированы в коде)

1. **Записи не отправляются на сервер** — сохраняются только в localStorage. Файл: `wizard.js` строка 386, комментарий `// В MVP сохраняем локально`.

2. **Слоты генерируются случайно** — функция `generateSlots()` в `data.js` генерирует псевдослучайные доступные слоты на каждую дату. Слоты могут различаться при каждом запуске.

3. **Прошедшие записи** — запись считается «прошедшей» по дате `dateIso`. Сразу после создания запись попадает в «Предстоящие».

4. **«Написать мастеру»** — открывает `https://t.me/alina_nails_msk`. Этот аккаунт — тестовые данные, не реальный контакт.

5. **«Поделиться с другом»** — в Telegram использует `switchInlineQuery`, в браузере открывает `t.me/share/url`.

### Технические ограничения

6. **Нет валидации телефона** — поле `type="tel"` без дополнительной проверки формата.

7. **Wizard не сохраняет шаг** — при закрытии и открытии приложения wizard сбрасывается.

8. **Галерея** — слайды — это цвета (`#A78BFA`, `#8B5CF6`...), реальных фото нет. Данные в `service.photos`.

## 6. Деплой (для тестировщика)

### Деплой на Vercel

**Требования:** Node.js, аккаунт Vercel (anluts@... — уточнить у владельца)

```bash
# Установить Vercel CLI
npm install -g vercel

# Войти

# Задеплоить из папки tg-app/
cd tg-app
vercel --prod --yes
```

> ⚠️ Если сеть блокирует Vercel API — использовать прокси:  
> `HTTPS_PROXY=http://191.102.148.189:9842 vercel --prod --yes`

**URL после деплоя:** `https://beauty-catalog-azure.vercel.app` (всегда один и тот же production alias)

### Обновить код и передеплоить

```bash
# Из корня репозитория
git add .
git commit -m "fix: описание изменений"
git push origin main

# Затем деплой
cd tg-app && vercel --prod --yes
```

## 7. Как отлаживать ошибки

### Отладка в браузере

1. Открыть `http://localhost:3000`
2. F12 → Console — все JS-ошибки
3. Application → Local Storage — состояние данных

### Отладка в Telegram

При JS-ошибке приложение показывает красный экран с:
- Текстом ошибки
- Именем файла
- Номером строки

Это обработчик `window.onerror` в `index.html` (в `<head>`).

### Сброс состояния для повторного теста

```js
// В консоли браузера — полный сброс:
localStorage.clear(); location.reload();

// Только онбординг:
localStorage.removeItem('onboarding_done_v1'); location.reload();

// Только оффер:
localStorage.removeItem('offer_shown_v1'); location.reload();

// Только записи:
localStorage.removeItem('beauty_bookings'); location.reload();
```

### В Telegram (DevTools недоступны)

Используй `@WebAppDebugBot` или подключи через `chrome://inspect` (Android):
1. Включить отладку: Telegram → Настройки → Advanced → Experimental → Enable WebView Debugging
2. На Android: Chrome → chrome://inspect → найти WebView
