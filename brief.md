# Brief: Telegram Mini App — Каталог услуг бьюти-мастера

> Версия: 1.0 (MVP)  
> Дата: 2026-04-07  
> Основа: research.md

---

## Концепция в одном предложении

Mini App для одного мастера или небольшой студии: клиентка открывает приложение из Telegram, листает работы, выбирает услугу и записывается — не выходя из мессенджера.

---

## Принципы дизайна

| Принцип | Что это значит |
|---------|---------------|
| Нативный Telegram | Цвета из `tg.themeParams`, `MainButton`, `BackButton`, `HapticFeedback` |
| Минимум текста | Фото говорит больше слов. Текст только там, где без него нельзя |
| Крупные элементы | Touch target ≥ 44×44px везде. Слоты времени — 48×64px |
| Русский язык | Весь интерфейс на русском |
| Приложение, не сайт | SPA, hash-роутинг, без перезагрузок, без хедеров с логотипом |
| Скорость | Skeleton UI сразу, данные подгружаются поверх. Цель: < 1.5с |

---

## Навигация верхнего уровня

```
[Каталог] [Мастер] [Записи]
   tab 1    tab 2    tab 3
```

Bottom tab bar всегда виден, 3 вкладки (не 5 — меньше перегруза для одного мастера).  
`MainButton` Telegram — основной CTA на каждом функциональном экране.

## Экраны: полное описание

---

### Экран 1 — Каталог (вкладка «Каталог»)

**Что видит пользователь:**

```
┌─────────────────────────────┐
│ 👋 Привет, [Имя]!           │  ← имя из Telegram
│ 🔍 Поиск услуг...           │  ← поисковая строка (не активная)
├─────────────────────────────┤
│ [Все] [Ногти] [Брови] [Лицо]│  ← горизонтальный скролл, chips
│ [Тело] [Волосы]             │
├─────────────────────────────┤
│ ┌──────────┐  ┌──────────┐  │
│ │  [фото]  │  │  [фото]  │  │  ← 2-колоночный грид
│ │ Название │  │ Название │  │
│ │ 60 мин   │  │ 90 мин   │  │
│ │ 1 800 ₽  │  │ 2 500 ₽  │  │
│ │ ★ 4.9    │  │ ★ 4.8    │  │
│ └──────────┘  └──────────┘  │
│ (повторяется...)            │
├─────────────────────────────┤
│ [Каталог]  [Мастер]  [Записи] │  ← bottom tab bar
└─────────────────────────────┘
```

**Элементы:**
- Приветствие с именем пользователя из Telegram (`first_name`)
- Поисковая строка (тап → Экран поиска)
- Горизонтальный скролл категорий: «Все», «Ногти», «Брови», «Лицо», «Тело», «Волосы»
- Карточки услуг 2 в ряд:
  - Квадратное фото (соотношение 1:1)
  - Название услуги (bold, 14px)
  - Длительность (серый, 12px)
  - Цена (accent цвет Telegram, 14px bold)
  - Рейтинг звёздочкой (если есть отзывы)
- Skeleton-карточки при загрузке

**Действия:**
- Тап по категории → фильтрует каталог (без навигации, inline)
- Тап по карточке услуги → Экран услуги
- Тап по поиску → Экран поиска

**MainButton:** скрыта на этом экране.

**Переходы:**
- Карточка → Экран 2 (Услуга)
- Поиск → Экран 6 (Поиск)
- Tab «Мастер» → Экран 3
- Tab «Записи» → Экран 4

│ ─────────────────────────── │
│ Что входит:                 │
│ ✓ Снятие покрытия           │
│ ✓ Уход за кутикулой         │
│ ✓ Покрытие гель-лаком       │
│ ✓ Топ-слой и сушка          │
│ ─────────────────────────── │
│ Отзывы                      │
│ ┌───────┐ ┌───────┐ →       │  ← горизонтальный скролл

**Элементы:**
- Свайпабельная галерея фото (до 5 фото), пагинация точками
- Название услуги
- Иконки с длительностью и ценой
- Рейтинг + количество отзывов (тап → прокрутка к отзывам)
- Список «Что входит» (3–5 пунктов с галочками)
- Горизонтальный скролл карточек отзывов:
  - Аватар + имя клиентки
  - Текст отзыва (2 строки)
  - Фото результата (если есть)
  - Дата
- `MainButton`: «Записаться · [цена] ₽»

**Действия:**
- Свайп галереи влево/вправо
- Тап по фото → лайтбокс (полноэкранный просмотр)
- Тап по отзывам → Экран отзывов (список)
- `MainButton` «Записаться» → Экран 5 (Wizard записи)
- `BackButton` → назад в Каталог

**Переходы:**
- MainButton → Экран 5 (Wizard, Шаг 1)
- BackButton → Экран 1 (Каталог)

  - Дата и время
  - Имя мастера + адрес

- «Записаться снова» → Экран 5 (Wizard, услуга и мастер предзаполнены)
- `MainButton` → Экран 1 (Каталог)

### Экран 5 — Wizard записи (3 шага)

#### Шаг 1: Выбор даты и времени

```
┌─────────────────────────────┐
│ ← [BackButton]              │
│ Маникюр классический ·1800₽ │  ← sticky заголовок (услуга)
│ ━━━━━━━░░░░░░░░  1 из 3    │  ← прогресс-бар

│ [7пн] [8вт] [9ср] [10чт]→  │  ← горизонтальный скролл дат
│  •          ✓               │  ← точка=есть слоты, серый=занято
├─────────────────────────────┤
│ Утро                        │
│ ┌────────┐ ┌────────┐       │
│ │ 10:00  │ │ 11:30  │       │  ← слоты, 3 в ряд
│ └────────┘ └────────┘       │
│ День                        │
│ ┌────────┐ ┌──────────┐     │
│ │ 14:00  │ │░16:00 ░░ │     │  ← серый = занят
│ └────────┘ └──────────┘     │
│ Вечер                       │
│ ┌────────┐ ┌────────┐       │
│ │ 18:00  │ │ 19:30  │       │
│ └────────┘ └────────┘       │
└─────────────────────────────┘
│     [ Выбрать время ]       │  ← MainButton (активна после выбора)
```

**Элементы:**
- Sticky-заголовок: название услуги + цена
- Прогресс-бар «1 из 3»
- Горизонтальный скролл дат (ближайшие 14 дней):
  - День недели + число
  - Точка под датой = есть свободные слоты
  - Серый = нет слотов / выходной
- Сетка слотов времени (3 в ряд, 48×64px):
  - Доступный: белый/карточный фон
  - Занятый: серый, некликабельный
  - Выбранный: цвет кнопки Telegram
  - Разделители «Утро / День / Вечер»
- `MainButton`: «Выбрать время» (неактивна до выбора)

**Действия:**
- Тап по дате → обновляет список слотов
- Тап по слоту → выделяет, активирует MainButton
- `MainButton` → Шаг 2
- `BackButton` → Экран услуги

**Элементы:**
- Горизонтальные табы-категории
- 3-колоночный грид квадратных фото
- Lazy loading при скролле

**Действия:**
- Тап по фото → лайтбокс (полноэкранный просмотр, свайп между фото)
- `MainButton` → Экран 5 (Wizard)
- `BackButton` → Экран Мастера

## Чего НЕ будет в первой версии (MVP)

### Функции, отложенные на V2

| Что | Почему отложено |
|-----|----------------|
| Корзина (несколько услуг) | Сложная логика пересчёта времени и слотов |
| Фото-отзывы клиенток | Требует хранилище и модерацию |
| Карта / геолокация | Для одного мастера избыточно |
| Предоплата (Telegram Payments) | Требует подключение платёжного провайдера |
| Онбординг / приветственный экран | Пользователи приходят по ссылке — уже «тёплые» |
| Система лояльности и баллы | Отдельная логика, не нужна для MVP |
| Умные напоминания (через 3 нед.) | Требует планировщик задач на бэкенде |
| Реферальная программа | Маркетинговая механика, не MVP |
| Поиск мастеров рядом | Это маркетплейс, а не приложение одного мастера |
| Мультиязычность | Только русский в MVP |
| Веб-версия (вне Telegram) | Фокус на Telegram Mini App |
| Расписание мастера для клиентки | Только доступные слоты, не полное расписание |
| Избранное / сохранённые услуги | Будет в V2 |
| Push-уведомления о новых слотах | Будет в V2 |
| Отзыв с фото | Будет в V2 |

### Намеренно упрощено в MVP

| Что | Как в MVP | Как в V2 |
|-----|-----------|---------|
| Отзывы | Текст + имя + дата | + фото результата |
| Уведомления | Только через бота в чат | + push за X часов |
| Отмена записи | `showConfirm` → запрос | + политика штрафов |
| Портфолио | Просмотр без фильтров | + фильтры по стилю |
| Личный кабинет | Только «Мои записи» | + профиль, настройки |

## Технический стек (для разработки)

```
Frontend:
  React 18 + TypeScript
  Vite (сборщик)
  Zustand (стейт wizard)
  TanStack Query v5 (кэш запросов)
  React Router v6 — HashRouter (#/)
  @twa-dev/sdk (типы для Telegram WebApp)
  dayjs (даты, 2KB)
  Tailwind CSS v4

Backend:
  Node.js / любой (REST API)
  Telegram Bot API (отправка подтверждений)

Изображения:
  WebP формат
  3 размера: thumb 150×150, preview 400×400, full 800×800
  Lazy loading через Intersection Observer

Хранилище:
  CloudStorage API — телефон и имя клиентки
  sessionStorage — стейт wizard (на случай сворачивания)
  TanStack Query cache — каталог и слоты
```
